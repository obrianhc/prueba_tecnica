# Política de Seguridad

Este repositorio se utiliza con fines educativos y de evaluación.

## Reglas de Oro

1. **Nunca ejecutes comandos que no entiendes.**
2. Inspecciona siempre los scripts definidos en `package.json` antes de ejecutarlos.

## Reporte de Vulnerabilidades

Durante la prueba, si encuentras una "vulnerabilidad" o comportamiento inesperado, tu tarea es identificarla, explicarla y, si es parte del ejercicio, corregirla o mitigarla.

El evaluador puede solicitarte ejecutar tareas específicas para comprobar tu atención a los detalles de seguridad.
